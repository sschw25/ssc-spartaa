# 썸머스쿨 모집 페이지 개편 및 캠퍼스별 적용 계획

SSC 스파르타의 썸머스쿨 페이지를 단순 안내가 아닌 **'직접적인 상담 및 예약 전환(Conversion)'**을 유도하는 랜딩 페이지로 전면 개편합니다. 관리형 학원의 정체성(완벽한 통제, 몰입)을 유지하면서, 학부모와 학생이 필요로 하는 모든 정보를 한 페이지에서 직관적으로 확인할 수 있도록 재구성합니다.

## User Review Required
> [!IMPORTANT]
> - **예약 링크 및 비용**: 모두 추후 입력하실 수 있도록 `#` 임시 링크 및 빈 공간(혹은 '상담 시 안내')으로 비워두고 UI만 먼저 잡아두겠습니다.

## Open Questions
> [!WARNING]
> - 질문에 모두 답변해 주셔서 추가 질문은 없습니다. 바로 개발을 진행해도 될지 확인만 부탁드립니다!

## Proposed Changes

썸머스쿨 관련 데이터를 중앙 집중화하여 관리하고, 공통 썸머스쿨 메인 페이지와 각 캠퍼스별 상세 페이지에 일관된 컴포넌트로 렌더링합니다.

### 1. 콘텐츠 데이터 구조 개편

썸머스쿨 페이지에 들어갈 텍스트와 데이터를 전면 교체 및 확장합니다.

#### [MODIFY] `lib/summer-content.ts`
- **Hero & Title**: "2026 SSC 스파르타 썸머스쿨 모집", "혼자서는 무너지는 여름방학을 관리로 바꾸는 시간" 등으로 메인 카피 변경
- **Overview (모집 개요)**: 대상, 기간, 운영센터, 모집인원, 개강일, 운영시간, 접수방법 데이터 추가
- **Features (프로그램 특징)**: 대치 썸머스쿨의 검증된 자료(매일 영단어 테스트, 국어/영어/수학 주간테스트)를 기반으로 한 학습 점검 및 관리 시스템 강조. (커넥츠프랩 문구 제외)
- **Schedule (하루 시간표 예시)**: 08:30 등원부터 21:30 마감까지의 기존 상세 타임라인 포맷 유지.
- **Recommended For (이런 학생에게 추천)**: 타겟 학생 리스트 추가 (방학 패턴 붕괴, 피드백 부족 등)
- **Registration Process (등록 절차)**: 예약 -> 상담 -> 진단 -> 좌석확인 -> 등록 확정의 5단계 프로세스 추가
- **Pricing & Inclusions (비용 및 포함/불포함 사항)**: 지정석, 출결관리 등 포함사항과 별도사항, 수강료(상담 후 안내) 데이터 추가
- **FAQ (자주 묻는 질문)**: 전달주신 Q&A 10가지 세트로 완전 교체
