# 여름방학 모집 페이지 개편 및 캠퍼스별 적용 계획

SSC 스파르타의 여름방학 페이지를 단순 안내가 아닌, 직접 상담과 예약 전환을 유도하는 랜딩 페이지로 개편합니다. 관리형 학원의 정체성인 환경, 통제, 몰입을 살리면서 학생과 학부모가 필요한 정보를 페이지 안에서 바로 확인할 수 있게 구성합니다.

## 사용자 확인 필요

> [!IMPORTANT]
> - 예약 링크와 비용: 모두 추후 입력할 수 있도록 `#` 임시 링크 및 빈 공간 또는 "상담 후 안내" 문구로 구성합니다.

## 열린 질문

> [!WARNING]
> - 현재 추가 질문은 없습니다. 바로 개발을 진행하되, 추후 확정 정보가 생기면 링크와 비용 문구만 교체하면 됩니다.

---

## 제안 변경 사항

여름방학 관련 데이터를 중앙 집중형으로 관리하고, 공통 여름방학 메인 페이지와 각 캠퍼스별 상세 페이지를 일관된 컴포넌트로 렌더링합니다.

### 1. 콘텐츠 데이터 구조 개편

여름방학 페이지에 들어갈 텍스트와 데이터를 전면 교체 및 확장합니다.

#### [MODIFY] `lib/summer-content.ts`

- **Hero & Title**: "2026 SSC 스파르타 여름방학 모집", "의지로는 무너지기 쉬운 여름방학을 관리로 바꾸는 시간" 등으로 메인 카피 변경
- **Overview (모집 개요)**: 대상, 기간, 운영 센터, 모집 인원, 개강일, 운영 시간, 접수 방법 데이터 추가
- **Features (프로그램 특징)**: 대치 여름방학에서 검증된 자료와 매일 상담, 국어/영어/수학 주간 테스트 기반 자기주도 학습 관리 시스템 강조
- **Schedule (하루 시간표 예시)**: 08:30 등원부터 21:30 마감까지의 상세 타임라인 정리
- **Recommended For (이런 학생에게 추천)**: 방학 패턴 붕괴, 피드백 부족, 학습 루틴 관리가 필요한 학생 리스트 추가
- **Registration Process (등록 절차)**: 예약 -> 상담 -> 진단 -> 좌석 확인 -> 등록 확정의 5단계 프로세스 추가
- **Pricing & Inclusions (비용 및 포함/별도 사항)**: 지정석, 출결 관리 등 포함 사항과 별도 사항, 수강료 "상담 후 안내" 데이터 추가
- **FAQ (자주 묻는 질문)**: 전달받은 Q&A 10개를 반영

## 검증 계획

### 수동 검증

1. `localhost:3000/wonju/summer`, `localhost:3000/chuncheon/summer`, `localhost:3000/chungju/summer` 경로에 각각 접속하여 페이지가 정상 렌더링되는지 확인합니다.
2. 컨텐츠프롬프트 문구 대신 대치 여름방학 자료, 매일 상담, 국어/영어/수학 주간 테스트 내용이 반영되었는지 확인합니다.
3. 예약 버튼 클릭 시 임시 링크(`#`)로 연결되는지 확인합니다.
4. 모바일 화면에서 시간표, 등록 절차 등의 가독성이 깨지지 않는지 반응형 UI를 테스트합니다.
