## 2. 시나리오별 사용 일지 요약

### 주 1 (6/1~6/7) — 시스템 온보딩 & 기본 업무

**A3 최서연 (신입 관리자)**
- 학생 상세 시트를 처음 열었을 때 탭이 6개나 있어서 어디서부터 볼지 혼란스러웠음
- 상담 기록 작성 후 저장 버튼을 누르지 않고 탭을 닫았더니 내용이 사라짐 → **자동저장 중인지 알 수 없음**
- 충주 캠퍼스 학생만 보고 싶은데 페이지 이동 시마다 캠퍼스 필터가 초기화됨

**S7 윤재원 (뽀모도로 첫 사용)**
- 뽀모도로 타이머를 켰다가 브라우저 탭을 전환했더니 타이머가 멈춤
- 25분 완료 후 쿠폰 지급 알림이 없어서 쿠폰이 쌓였는지 모름
- 홈 탭의 "오늘 할 일" 카드에서 체크한 항목이 새로고침하면 초기화됨 (P1 버그)

**S8 강민서 (모바일 이용자)**
- 실행 계획 탭에서 주간 계획이 7열 그리드로 표시되어 모바일에서 세로로 길게 스크롤해야 함
- 글자가 너무 작고 터치 영역이 협소함

---

### 주 2 (6/8~6/14) — 상담 & 진도 관리 집중

**A1 박지윤 (미션 정산)**
- 월간 미션 설정을 변경하다가 실수로 다른 페이지로 이동 → 변경 내용 전부 날아감 **(이탈 경고 없음, P1)**
- 정산 버튼 클릭 시 confirm 창이 2번 연속 떠서 당황스러움
- 마지막 정산 시점을 확인할 방법 없음 → 이중 정산 우려

**A2 이민준 (출결 관리)**
- 토요일 날짜 계산이 간혹 하루 빠르게 표시되는 현상 (UTC/KST 시간대 불일치)
- 출결 시간 수동 수정 중 날짜를 바꿨더니 이전 날짜의 수정 내용이 새 날짜에도 남아있음
- 외출 상태 학생의 복귀 처리가 누락되어도 시스템에서 알림 없음

**S4 정민아 (학부모 공유 사용)**
- 학부모 공유 링크를 생성했는데 링크를 본인이 어디서 확인하는지 안내가 없음
- 관리자에게 "링크 어디있어요?" 문의

**S9 임태양 (진도 변경 신청)**
- 진도 변경 신청 폼의 "메시지" 칸이 비어있어도 전송 버튼이 활성화됨
- 빈 내용으로 신청이 전송되어 관리자 혼란 발생 **(P1 버그)**

---

### 주 3 (6/15~6/21) — 휴가·벌점·모의고사 집중 사용

**A2 이민준 (휴가 신청 처리)**
- 반차 쿠폰 잔여량이 0인 학생에게 쿠폰을 차감 클릭했더니 API 에러 후 아무 반응 없음
- "이달 반차 2회 소진" 메시지가 어디 표시되는지 찾기 어려움
- 쿠폰 페이지 제목이 "휴가 쿠폰 관리"인데 신청 목록도 있어서 기능 경계가 모호함

**S2 이하은 (반차 신청)**
- 반차 신청 시 날짜를 입력하지 않고 신청 버튼을 클릭하면 그냥 넘어감 (서버에서만 에러)
- 신청 후 상태를 어디서 확인하는지 처음에 몰랐음 (상담 탭 하단에 있음 - 탭 간 이동 필요)
- 달 선택기의 "2026. 06월" 표기가 어색함 → "2026년 6월"이 맞음 **(P2 버그)**

**S5 한승호 (벌점 누적 학생)**
- 벌점 내역이 최신순인지 오래된 순인지 정렬 기준 표시 없음
- 순공 리더보드에서 자신의 순위를 확인하고 싶은데 리더보드가 자동 갱신되지 않음

**S6 오지현 (모의고사 참여 응답)**
- 모의고사 불참 사유 제출 후 "처리 중" 상태가 언제 처리될지 기간 안내 없음
- 불참 승인됐는지 알림이 오지 않음

---

### 주 4 (6/22~6/29) — 리뷰 & 정산 마감

**A1 박지윤 (월말 정산)**
- 인박스에서 OT 불참 신청이 학생별로 별도 표시되어야 하는데 같은 eventId를 가진 항목들이 충돌하여 일괄 처리 시 엉뚱한 학생 처리 **(P1 버그 - 데이터 무결성)**
- 답변 초안을 입력하다가 다른 항목 클릭하면 초안이 사라짐 **(P1 버그)**

**A3 최서연 (리더보드 확인)**
- 리더보드 페이지가 마지막 조회 이후 갱신 안 됨 → 수동 새로고침 필요
- 캠퍼스별 학생 수가 몇 명인지 버튼에 표시 없음

**S10 노예린 (미션 쿠폰 최다 보유)**
- 쿠폰 잔액이 몇 개인지 홈 탭 미션 카드에서 바로 보이지만 어디서 쓸 수 있는지 안내 없음
- "3개 = 반차 1회 추가" 설명이 미션 카드에 없음 → 처음엔 쿠폰이 그냥 포인트인 줄 알았음

## 6. 기술 부채 항목 (추가 관찰)

### 6-1. KST 시간대 일관성
**현황**: `new Date()`, `getMonth()`, `getDate()` 등이 서버·클라이언트 모두 로컬 시간대 사용.  
**위험**: 한국 자정(UTC+9) 전후 1시간 동안 날짜 경계 오표시.  
**권고**: `date-fns-tz` 또는 커스텀 `toKST()` 유틸 함수를 작성하고 날짜 처리 코드를 일괄 교체.

### 6-2. window.confirm 사용
**현황**: 8개 이상의 파일에서 `window.confirm` 사용. iOS 15+ 에서 차단되는 브라우저 있음.  
**권고**: 프로젝트 내 `ConfirmDialog` 컴포넌트 단일화 후 교체.

### 6-3. God 컴포넌트
**현황**: `student-detail-sheet.tsx` ~4,300줄, `use-report-state.ts` 무거운 계산 미캐싱.  
**권고**: 탭별 lazy load + 도메인 훅 분리 (LAUNCH_CHECKLIST 항목 이행).

### 6-4. 인박스 항목 고유 ID 설계
**현황**: 이벤트 기반 항목(`ot:eventId`, `exam:examId`)이 학생 ID를 포함하지 않음.  
**권고**: 복합 키 `{type}:{eventId}:{studentId}` 사용.

### 6-5. 빈 상태 UI 패턴 부재
**현황**: 필터 결과 0건, 데이터 미입력 상태에서 빈 화면 방치.  
**권고**: `<EmptyState icon variant message>` 컴포넌트 제작 후 전역 적용.

## 7. 긍정 피드백

시뮬레이션 중 사용성이 우수하다는 평가를 받은 항목:

- **iOS26 Liquid Glass 디자인**: 모든 페르소나에서 "고급스럽다", "깔끔하다" 반응
