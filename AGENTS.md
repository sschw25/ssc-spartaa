## 🎨 색상 시스템 — "색 = 위계/의미" (가장 중요)

**대원칙: 색은 장식이 아니라 의미다.** 카드·지표·뱃지마다 "다양해 보이려고" 임의의 색을
칠하지 말 것. 색이 다르면 **의미가 달라야** 한다. 같은 성격의 정보(예: 정보성 숫자)는 같은 색.

### 사용 가능한 색 — 의미색 4계열 + 중립뿐
| 색 | 토큰 | 의미 | 예시 |
|---|---|---|---|
| 🔵 파랑 | `#0071E3` / `blue-600` | 정보 · 주요 · 클릭 가능 | 총 원생, 평균 진도율, 실시간 몰입 |
| 🟢 초록 | `emerald-*` / `#34C759` | 양호 · 등원 · 완료(긍정 상태) | 등원중, 누적 학습, 출석 |
| 🟠 앰버/오렌지 | `amber-*` / `#F56300` | 주의 · 조치 필요 | 성적 미입력, 미학습, 진도 부족, 상담 도래, 지각 |
| 🔴 빨강 | `red-*` | 위험 · 실패 · 미등원 | 결석, 등록 만료, 처리 실패 |
| ⚪ 중립 | `#1D1D1F`(잉크) · `#86868B`(보조) · `slate-*` | 기본 텍스트·수치, 분류 라벨 | 본문, 일반 수치, 캠퍼스 태그 |

### 금지 사항
- ❌ **보라(purple/violet)·인디고(indigo) 금지.** 과거 KPI·타입색으로 남발되어 전부 제거함. 다시 넣지 말 것.
- ❌ **하드코딩 보라 `#862bf7` 금지.** (교재/인강 구분에 쓰였으나 파랑으로 통일함)
- ❌ **존재하지 않는 Tailwind 셰이드 금지:** `-450 / -650 / -750 / -850` 등은 클래스가 생성되지 않아 **무효**(색이 안 먹고 기본 검정으로 렌더)된다. 유효 셰이드(`50,100,200,...,600,700,800,900`)만 사용.
- ❌ 같은 성격의 지표에 카드마다 다른 색을 칠해 "컬러풀하게" 만들지 말 것.

### 분류(nominal)는 색이 아니라 텍스트·아이콘으로 구분
- **캠퍼스**(원주/춘천/충주): 의미색과 충돌하므로 **중립 회색 뱃지로 통일**하고 텍스트로 식별.
- **교재 vs 인강**: 색 구분 금지. 📚(교재)·💻(인강) **아이콘 + 파랑**으로 통일.

### 위계 예시 (대시보드)
- 총 원생·평균 진도율 = **파랑(정보)** → 참고용으로 가라앉음
- 성적 미입력·상담 도래 = **앰버(주의)** → "조치 필요"로 시선이 먼저 감
- 색의 개수가 아니라 **의미의 명확함**이 목표. 의심되면 중립(회색)이 기본값.

## 🧩 CSS / 스타일 함정 (실제로 사고가 났던 부분)

- **전역 CSS는 `app/globals.css` 하나만 살아있다.** `styles/globals.css`는 **죽은 파일**(어디서도 import 안 됨, shadcn 기본값 잔재). 공용 유틸/클래스/키프레임은 **반드시 `app/globals.css`**에 추가할 것. (과거 `transition-premium`·`animate-fade-in-up`을 죽은 파일에 넣어 전부 무효였던 사고 있었음)
- **어드민 앱 셸 배경색 = `#F8F9FA`** 로 통일 (대시보드/상담/랭킹/출결의 `min-h-screen` 루트). 칩·인풋 등 작은 채움색은 `#F5F5F7`.
- 디자인 톤: **Apple-light** — `--radius: 0.75rem`, 흰 카드 + 옅은 그림자, Pretendard 폰트, `#1D1D1F` 텍스트.
